### ⚠ Partial (BLOCKER)

**STATUS-06 — persistSubmission summary shape mismatch.** This is the only functional wiring gap in v1.2.

- **Location:** `src/pages/feedback.astro:272-278`
- **What's wrong:** Both branches of `persistSubmission()` read `.locator.pageRoute`:

  ```js
  // v2 branch (line 273):
  var r = e && e.locator && e.locator.pageRoute;

  // v1 branch (line 276):
  else if (payload && payload.locator && payload.locator.pageRoute) { ... }
  ```

- **Why it doesn't work:** `public/feedback-inject.js` emits a FLAT payload on both paths.
  - v1 `buildPayload()` (line 691-696) returns `{ schemaVersion, pageRoute, astroFileGuess, intent, ... }` — `pageRoute` is at the top level, NO `.locator` wrapper.
  - v2 batch `editObj` (line 1015-1025) explicitly spreads locator fields flat to mirror v1 (per source comment: *"Spread locator fields (mirrors v1 buildPayload's flat shape so validateEdit on the server sees the same per-edit object shape)."*).

- **Observable impact:**
  - v2 batch submission → row summary reads `"3 edits"` (count correct) but no route list appended.
  - v1 single submission → row summary reads `"0 edits"` because the entire `else-if` guard is false, so `editCount` is never incremented from 0.

- **Two-line fix:**
  ```diff
  -                  var r = e && e.locator && e.locator.pageRoute;
  +                  var r = e && e.pageRoute;
  ```
  ```diff
  -                } else if (payload && payload.locator && payload.locator.pageRoute) {
  +                } else if (payload && payload.pageRoute) {
                     editCount = 1;
  -                  routes.push(payload.locator.pageRoute);
  +                  routes.push(payload.pageRoute);
                   }
  ```
