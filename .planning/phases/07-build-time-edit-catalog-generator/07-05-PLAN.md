<!-- Existing v1.1/v1.2 sanity-script patterns to mirror. -->

The repo already has post-build oriented scripts under `scripts/`:
  - scripts/check-public-clean.mjs — fails the build if forbidden directories exist (runs as `prebuild`)
  - scripts/canary.sh — wraps the smoke harnesses

`scripts/check-edit-catalogs.mjs` follows the same patterns:
  - process.exit(1) with a clear message on assertion failure
  - process.exit(0) silently on success (matches CI norms)
  - Logs progress with console.log only on success summary

Catalog schema to validate (from 07-02 + 07-03):

  {
    buildSha: string  // 6-12 hex chars, /^[0-9a-f]{6,12}$/ — 'unknown' is a hard failure
    route: string     // must start with '/'
    generatedAt: string  // ISO-8601
    entries: Array<{
      id: string                  // /^[0-9a-f]{12}$/
      kind: 'i18n-text' | 'i18n-html' | 'image' | 'gallery-image' | 'heading' | 'hardcoded-text'
      // ... locator-signal fields per kind
      requiresManualSelection?: boolean
    }>
  }

Sanity-script checks:
  1. find dist/edit-catalogs -name '*.json' produces >= 8 files
  2. Every file parses as JSON
  3. Every file has all 4 top-level keys
  4. buildSha matches /^[0-9a-f]{6,12}$/ — the script REJECTS 'unknown' as a failure (SC#4 requires a real SHA, and Vercel deploys always have .git so 'unknown' is unreachable in production paths)
  5. entries is non-empty for the home page + at least one /homes/* page (sanity that the walker actually walked)
  6. Per-entry: id matches /^[0-9a-f]{12}$/, kind is one of the 6 enum values
  7. Cross-entry: ids are unique within a catalog
  8. .vercelignore does NOT contain a rule matching dist/edit-catalogs (CATALOG-06 ship-to-prod guard)
