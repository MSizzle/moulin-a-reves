The new editCatalog integration shape (Astro integration contract):

  // src/integrations/edit-catalog/index.mjs
  export default function editCatalog() {
    return {
      name: 'edit-catalog',
      hooks: {
        'astro:build:done': async ({ dir, routes, pages, logger }) => {
          // dir: URL of the output directory (e.g. file:///path/to/dist/)
          // routes: array of RouteData; routes[].pathname is '/' or '/homes/le-moulin' etc.
          // pages: array of { pathname: string } — alternate iteration surface
          // logger: Astro's logger
        },
      },
    };
  }

Prerendered routes:
  - routes[i].prerender === true (or pages[].pathname) marks static HTML output.
  - Filter to prerender === true; non-prerendered routes are Vercel Functions (no HTML on disk).

Route → catalog filename mapping:
  - '/' → dist/edit-catalogs/index.json
  - '/homes/le-moulin' or '/homes/le-moulin/' → dist/edit-catalogs/homes/le-moulin.json
  - '/the-compound' → dist/edit-catalogs/the-compound.json
  - Strip leading slash, strip trailing slash, '' → 'index', append '.json'

Astro emits HTML at:
  - dist/index.html for '/'
  - dist/homes/le-moulin/index.html for '/homes/le-moulin' (Astro 6 default directory-style output)
  - The walker (07-03) will read this HTML via path.join(dir.pathname, route.pathname, 'index.html').
