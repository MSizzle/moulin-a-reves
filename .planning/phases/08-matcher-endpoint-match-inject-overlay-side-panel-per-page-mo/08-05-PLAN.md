Complete the per-page review UI by rendering the side panel inside the per-page mode pane and wiring all five PANEL-* requirements:

1. **PANEL-01** — Render one row per match with line text, primary match preview, confidence pill (green/amber/red/no-match tier per D-06), alternates list (with `<code>` ID chips), italic reason (truncated at 120 chars per D-19), and Approve/Reject/Pick-manually buttons.

3. **PANEL-03** — Pick-manually clears the row's pin via `mar:feedback:match-pick-manually` postMessage and surfaces the in-iframe toast (Plan 03 handles the toast). Inbound `mar:feedback:manual-resolved` updates the row's preview.

5. **PANEL-05** — `rowStates` persists in sessionStorage so panel state survives iframe navigation; clears on browser close.

Also wire the catalog-drift banner (D-15..D-17) and the message listener for inbound `mar:feedback:*` messages from the inject (Plan 03).

matchSet sessionStorage shape (already written by Plan 04):
  { id, route, buildSha, createdAt, editList, matches: Match[], rowStates: RowState[] }

Match (Plan 02 endpoint response shape, per MATCH-04):

RowState (Plan 04 initializes; Plan 05 mutates):
  { lineIndex: number, status: 'pending' | 'approved' | 'rejected' | 'manual', manualLocator: object | null }

v2 staged-edit shape (target of PANEL-02 Approve write):
  The v1.1 corner chip writes into sessionStorage['mar_feedback_staged_v1'] as a JSON-serialized array. Each element has the shape produced by feedback-inject.js's captureLocator() — see .github/CLAUDE_FEEDBACK.md §8 for the canonical schema. Required fields for the locator (validated by src/pages/api/feedback/validate.ts):
    {
      schemaVersion: 2,
      pageRoute: string,            // e.g. '/homes/le-moulin/'
      action: 'edit-text' | 'edit-image' | 'edit-html' | 'unknown',
      i18nKey?: string,             // for i18n-text / i18n-html / i18n-placeholder kinds
      i18nAttr?: string,            // 'data-i18n' | 'data-i18n-html' | 'data-i18n-placeholder'
      currentText?: string,
      currentTextFr?: string,
      nearestHeading?: string,
      imageRef?: string,            // for image kinds
      altText?: string,
      domPath?: string,             // for hardcoded-text / heading fallback
      note: string,                 // the user's edit instruction (the match line text)
      stagedAt: string,             // ISO timestamp
      submitId?: string,            // optional unique id within the batch (uuid)
    }
  The validator requires signalCount(locator) >= 2 — at least two of {i18nKey, imageRef, domPath, nearestHeading, currentText} must be present. The catalog entries ALREADY carry these signals (Phase 7 walker uses the same locator-signals helper as feedback-inject.js), so the Approve handler derives them by reading the catalog entry matching primaryId.

Catalog fetch path (canonical — same slug logic as Plan 04 routeToCatalogUrl and Plan 02 loadCatalog):
  /edit-catalogs/<route-slug>.json
  where <route-slug> is route with leading/trailing slashes stripped, e.g. '/homes/le-moulin/' -> 'homes/le-moulin', and the bare '/' -> 'index'.

Inbound message types (from inject in Plan 03):
- mar:feedback:matchset-stale { matchSetId } -> show drift banner, disable Approve buttons
- mar:feedback:match-ready { matchSetId, paintedCount, totalCount } -> optional UX (no required action)
- mar:feedback:manual-resolved { lineIndex, locator } -> update row.status to 'manual', set rowStates[i].manualLocator = locator

Outbound message types (panel -> inject):
- mar:feedback:match-focus { lineIndex } -> on row body click
- mar:feedback:match-pick-manually { lineIndex } -> on Pick-manually button click

    function buildStagedEditFromCatalogEntry(entry, route, lineText) {
      // Mirror v1.1 captureLocator's output shape (.github/CLAUDE_FEEDBACK.md §8 + validate.ts signalCount).
      // The catalog already carries the locator signals — we just re-shape them into the staged-edit format.
      var staged = {
        schemaVersion: 2,
        pageRoute: route,
        action: 'unknown',
        note: lineText,
        stagedAt: new Date().toISOString(),
        submitId: 'pp_' + (globalThis.crypto && globalThis.crypto.randomUUID ? globalThis.crypto.randomUUID() : (Date.now() + '_' + Math.random().toString(36).slice(2, 10))),
      };
      // Action derivation by kind:
      if (entry.kind === 'image' || entry.kind === 'gallery-image') staged.action = 'edit-image';
      else if (entry.kind === 'i18n-html') staged.action = 'edit-html';
      else if (entry.kind === 'i18n-text' || entry.kind === 'heading' || entry.kind === 'hardcoded-text') staged.action = 'edit-text';
      // Locator signals — copy verbatim if present (catalog uses same field names as validate.ts).
      if (entry.i18nKey) staged.i18nKey = entry.i18nKey;
      if (entry.i18nAttr) staged.i18nAttr = entry.i18nAttr;
      if (entry.currentText) staged.currentText = entry.currentText;
      if (entry.currentTextFr) staged.currentTextFr = entry.currentTextFr;
      if (entry.nearestHeading) staged.nearestHeading = entry.nearestHeading;
      if (entry.imageRef) staged.imageRef = entry.imageRef;
      if (entry.altText) staged.altText = entry.altText;
      if (entry.domPath) staged.domPath = entry.domPath;
      return staged;
    }
    
    async function handleApprove(i) {
      var ms = loadMatchSet();
      if (!ms || !ms.matches[i]) return;
      var m = ms.matches[i];
      var rowState = ms.rowStates[i] || { lineIndex: i, status: 'pending', manualLocator: null };
      var catalog = await ensureCatalog(ms.route);
      var staged;
      if (rowState.manualLocator) {
        // Manual locator was captured via v1.1 click flow; use it directly (it's already in staged-edit shape).
        staged = Object.assign({}, rowState.manualLocator, { note: m.line, stagedAt: new Date().toISOString() });
      } else {
        if (!m.primaryId || !catalog) return;
        var entry = catalog.entries.find(function (e) { return e.id === m.primaryId; });
        if (!entry) return;
        staged = buildStagedEditFromCatalogEntry(entry, ms.route, m.line);
      }
      // Append to v1.1 corner chip's batch store (D-05 / D-24).
      var arr = loadStaged();
      arr.push(staged);
      saveStaged(arr);
      // Update local row state.
      rowState.status = 'approved';
      ms.rowStates[i] = rowState;
      saveMatchSet(ms);
      hasShownHelperThisSession = true;
      // Polite announcement for screen readers.
      announce('Match ' + (i + 1) + ' approved and staged for batch submission.');
      renderPanel();
    }
    
    function handleUndo(i) {
      // Remove the matching staged edit (by submitId or by route+note pair if submitId missing).
      var ms = loadMatchSet();
      if (!ms || !ms.matches[i]) return;
      var lineText = ms.matches[i].line;
      var arr = loadStaged();
      // Find the LAST staged edit matching this pageRoute + note (most-recent wins on dupes).
      for (var j = arr.length - 1; j >= 0; j--) {
        if (arr[j].pageRoute === ms.route && arr[j].note === lineText) { arr.splice(j, 1); break; }
      }
      saveStaged(arr);
      ms.rowStates[i].status = 'pending';
      saveMatchSet(ms);
      renderPanel();
    }
    
    function handleReject(i) {
      var ms = loadMatchSet();
      if (!ms || !ms.rowStates[i]) return;
      ms.rowStates[i].status = 'rejected';
      saveMatchSet(ms);
      announce('Match ' + (i + 1) + ' rejected. ' + countByStatus(ms, 'rejected') + ' total rejected.');
      renderPanel();
    }
    
    function handleRestore(i) {
      var ms = loadMatchSet();
      if (!ms || !ms.rowStates[i]) return;
      ms.rowStates[i].status = 'pending';
      saveMatchSet(ms);
      renderPanel();
    }

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05-01 | Tampering | Crafted catalog entry forging a staged edit that bypasses validate.ts | mitigate | `buildStagedEditFromCatalogEntry` copies entry fields verbatim into the staged-edit shape. | T-08-05-02 | Tampering | XSS via match.line, match.reason, or catalog entry text rendered into the panel | mitigate | Every text-rendering call uses `textContent` (not `innerHTML`). All match/entry text is set via `el.textContent = value` which auto-escapes. |
| T-08-05-03 | Tampering | Forged inbound `mar:feedback:manual-resolved` message with malicious locator | mitigate | The inbound dispatcher checks `ev.origin === location.origin` first. | T-08-05-05 | Denial of Service | Massive matchSet.matches array (>150 rows) slows panel render | mitigate | Plan 02's MATCH-07 cap (150 entries) prevents this server-side.
